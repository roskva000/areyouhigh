## 2024-05-24 - [Fix JSON-LD XSS Vulnerability]
**Vulnerability:** XSS (Cross-Site Scripting) vulnerability found in `src/components/SEO.jsx`. `JSON.stringify(jsonLd)` was rendered directly inside a `<script type="application/ld+json">` block. If `jsonLd` contained a malicious string like `</script><script>alert('XSS')</script>`, it could break out of the JSON context and execute arbitrary JavaScript.
**Learning:** React Helmet (and React in general) does not automatically escape `<` or `>` when stringifying JSON inside a script tag's text node, creating an XSS vector if data originates from an untrusted source or user input (e.g. dynamic content from a CMS).
**Prevention:** Always serialize JSON for insertion into HTML by replacing the `<` character with `\u003c` and inject using `dangerouslySetInnerHTML={{ __html: ... }}`. This prevents any possibility of injecting an HTML tag or breaking out of the `<script>` block.
